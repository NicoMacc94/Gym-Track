# Feature Plan – Homepage

## Obiettivo
Impostare la homepage `/` con titolo “Gym Track”, un sottotitolo che spiega lo scopo dell’app, e un placeholder statico per la futura lista degli allenamenti.

## Ambito
- **Incluso**: aggiornare `web/src/app/page.tsx` e relativi stili (`web/src/app/page.module.css` o `globals.css` se serve) per mostrare titolo, sottotitolo e testo placeholder (“Qui in futuro vedrai i tuoi allenamenti”).
- **Escluso**: funzionalità dinamiche, integrazioni dati, routing aggiuntivo, autenticazione, UI kit esterni.

## Implementazione proposta
- **Layout**: struttura a colonna centrata; hero con titolo e sottotitolo; sezione placeholder sotto.
- **Copie**: titolo “Gym Track”; sottotitolo esplicativo breve (es. “Monitora i tuoi allenamenti e progressi in un unico posto”); placeholder testuale indicato sopra.
- **Stili**: usare CSS module esistente (`page.module.css`) per spacing e tipografia semplice; nessun componente aggiuntivo.
- **Accessibilità**: titolo in `<h1>`, sottotitolo in `<p>` o `<h2>`, placeholder in `<section>` con copy chiara.

## Considerazioni
- Mantenere import alias `@/*` se necessario, ma probabile uso di percorsi relativi semplici.
- Non introdurre nuove dipendenze o modifiche a configurazioni.

## Piano di verifica
- `cd web && npm run lint`.
- Avvio rapido: `cd web && npm run dev` e check visivo della homepage.

## Esito implementazione
- Aggiornata la homepage con hero “Gym Track”, sottotitolo descrittivo e sezione placeholder “Qui in futuro vedrai i tuoi allenamenti” (`web/src/app/page.tsx`).
- Rinfrescato lo stile con layout centrato, card chiara e placeholder evidenziato (`web/src/app/page.module.css`).
- Verifica: `cd web && npm run lint` (ok).
